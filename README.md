# Hibernate Accessor Enhancer

An ASM-based bytecode enhancer that generates optimized field accessor inner classes for JPA/Hibernate entities. Instead of relying on reflection at runtime, the enhancer produces static inner classes that read and write entity fields via direct `GETFIELD`/`PUTFIELD` instructions.

## How It Works

The enhancer processes compiled `.class` files through a **two-pass pipeline**:

1. **Pass 1 -- Field Collection** (`PersistentFieldCollector`): scans the class bytecode and collects metadata about persistent fields. By default only JPA-annotated fields are collected (`@Id`, `@Basic`, `@ManyToMany`, `@OneToMany`, `@OneToOne`, `@ManyToOne`, `@ElementCollection`, `@Embedded`, `@EmbeddedId`, `@Version`, `@Lob`, and Hibernate's `@NaturalId`). The `--all-fields` flag includes every non-static, non-transient field unless it carries `@Transient`.

2. **Pass 2 -- Code Generation** (`AccessorWriterClassVisitor`): for each persistent field, generates two inner classes:
   - A **writer** implementing `org.hibernate.accessor.HibernateAccessorValueWriter` with a `set(Object, Object)` method.
   - A **reader** implementing `org.hibernate.accessor.HibernateAccessorValueReader` with a `get(Object)` method.

   Each inner class exposes a public `static final INSTANCE` singleton and registers itself in `HibernateAccessorFactory`'s cache during class loading.

### Generated Class Naming

For a field `isbn` in class `Book`:

```
Book$$__HibernateAccessorValueWriter__Book__isbn   (writer)
Book$$__HibernateAccessorValueReader__Book__isbn   (reader)
```

### Inheritance Support

The enhancer processes each class independently. Superclass accessors work on subclass instances because the generated bytecode uses `CHECKCAST` to the declaring class -- a subclass instance passes the check since it IS-A superclass.

## Requirements

- Java 11+
- Maven 3.x
- [hibernate-accessor-core](https://github.com/hibernate/hibernate-accessor-core) 0.0.1-SNAPSHOT

## Build

```bash
mvn clean package
```

This produces an executable uber-JAR via the Maven Shade Plugin.

## Usage

```bash
java -jar hibernate-accessor-enhancer.jar --input <classes-dir> --output <output-dir> [--all-fields]
```

| Flag | Description |
|---|---|
| `--input <path>` | Directory containing the compiled `.class` files to enhance |
| `--output <path>` | Directory where enhanced class files and generated inner classes are written |
| `--all-fields` | Process all non-static, non-transient fields (default: only JPA-annotated fields) |

### Example

```bash
java -jar hibernate-accessor-enhancer.jar \
  --input target/classes \
  --output target/enhanced-classes
```

## Tests

Run all tests:

```bash
mvn test
```

The test suite covers three areas:

- **`TestBytecodeEnhancement`** -- Verifies the bytecode structure of generated inner classes: correct interface implementation, `INSTANCE` field, `set()`/`get()` methods, static initializer, cache registration, and `PUTFIELD`/`CHECKCAST` instructions.

- **`TestEnhancedAccessors`** -- Functional tests that enhance `Book` and `Author` entities at runtime, load the generated accessors via a custom `ClassLoader`, and verify that `set()` actually writes field values and `get()` returns correct values for `String`, `LocalDate`, `Long`, and `Set` fields.

- **`TestEnhancedAccessorsWithInheritance`** -- Tests that superclass field accessors work correctly on subclass instances. Enhances `BasePublication` and `Novel` (extends `BasePublication`) and verifies that `BasePublication`'s `id` and `title` accessors can set/get values on a `Novel` instance.

## Project Structure

```
src/main/java/org/hibernate/accessor/bytecode/
    AccessorWriterClassVisitor.java   -- Pass 2: generates inner class bytecode
    FieldMetadata.java                -- Holds field name, descriptor, and type info
    HibernateAccessorEnhancerCli.java -- CLI entry point
    InnerClassGenerator.java          -- Emits writer/reader bytecode via ASM
    JpaAnnotationDetector.java        -- Recognizes JPA persistence annotations
    PersistentFieldCollector.java     -- Pass 1: collects persistent field metadata
    TypeDescriptorHelper.java         -- JVM type descriptor utilities

src/test/java/org/hibernate/accessor/bytecode/
    Book.java, Author.java                      -- Test entities
    BasePublication.java, Novel.java             -- Test entities (inheritance)
    TestBytecodeEnhancement.java                 -- Bytecode structure tests
    TestEnhancedAccessors.java                   -- Functional accessor tests
    TestEnhancedAccessorsWithInheritance.java    -- Inheritance accessor tests
```
